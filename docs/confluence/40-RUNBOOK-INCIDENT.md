# Runbook: Incident

## Trigger
Når denne runbooken brukes.

## Severity
- SEV1: Kritisk produksjonsbrudd
- SEV2: Delvis nedetid
- SEV3: Begrenset påvirkning

## Steg
1. Bekreft omfang og påvirkning
2. Opprett incident-ticket
3. Kommuniser status
4. Mitiger (rollback/feature flag/hotfix)
5. Verifiser gjenoppretting
6. Kjør postmortem

## Kommunikasjonsmal
- Hva skjer
- Hvem påvirkes
- Neste oppdateringstidspunkt

## Postmortem data
- Rotårsak
- Tidslinje
- Tiltak

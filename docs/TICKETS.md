# Tickets and Board

Dette repoet bruker GitHub Issues som ticketsystem.

## Anbefalt oppsett i GitHub UI

1. Gå til repository `Projects`.
2. Opprett et nytt prosjekt (Board).
3. Lag kolonner:
- Backlog
- Ready
- In Progress
- In Review
- Done

## Issue-typer

- Bug: produksjonsfeil eller funksjonell feil.
- Feature: ny funksjonalitet.
- Task: avgrenset arbeidspakke.

## Enkle arbeidsregler

- Alle endringer starter som issue.
- PR skal linke til issue (`Closes #<id>`).
- Issue lukkes først når DoD er oppfylt og kode er merg'et.

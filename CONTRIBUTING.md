# CONTRIBUTING

## Arbeidsmodell

- Alt arbeid skjer via branch + pull request.
- Endringer skal være små, målrettede og testbare.
- Dokumentasjon oppdateres i samme endring som kode.

## Branching

- `main`: stabil hovedgren.
- Feature-branches: `feat/<kort-beskrivelse>`.
- Fix-branches: `fix/<kort-beskrivelse>`.

## Pull requests

- Beskriv hva som er endret og hvorfor.
- Knytt PR til relevant issue (når issue finnes).
- Legg ved teststatus og eventuelle risikopunkter.

## Kvalitetskrav

- Lint/typecheck/test skal passere.
- Ingen secrets i repo.
- Nye filer skal ha tydelig formål.

## Dokumentasjonskrav

- Oppdater `docs/WORKLOG.md` ved alle vesentlige endringer.
- Oppdater `README.md` når struktur eller oppstart endres.
